# Packaging and Public Release

Use this checklist before publishing the skill as a public GitHub repository or release archive.

## Include

```text
nature-figure-learner/
├── SKILL.md
├── README.md
├── INSTALL.md
├── PACKAGING.md
├── AGENT_COMPATIBILITY.md
├── CHANGELOG.md
├── CONTRIBUTING.md
├── LICENSE
├── agents/
├── references/
└── scripts/
```

## Exclude

Do not publish:

```text
__pycache__/
*.pyc
figure-kb/
real KB data
paper PDFs
paper screenshots
private images
private paths
tokens or credentials
backup-*/
```

The `.gitignore` file excludes these by default.

## Pre-Release Verification

From the repository root:

```bash
python -B -m py_compile scripts/kb_location_manager.py scripts/self_evolution_engine.py scripts/test_self_evolution_engine.py
python -B -m unittest scripts/test_self_evolution_engine.py
```

Check for private paths or secrets:

```bash
rg "C:\\Users\\|/Users/|figure-kb/index.json|token|secret|api_key" .
```

Inspect what will be committed:

```bash
git status
git add --dry-run .
git ls-files
```

## Create a GitHub Repository

Initialize locally:

```bash
git init
git add .
git commit -m "Initial public release"
```

Create and push with GitHub CLI:

```bash
gh repo create nature-figure-learner --public --source . --remote origin --push
```

Or create the repository on GitHub first, then push:

```bash
git remote add origin https://github.com/<owner>/nature-figure-learner.git
git branch -M main
git push -u origin main
```

## Release Archive

From the parent directory:

```powershell
Compress-Archive -Path nature-figure-learner -DestinationPath nature-figure-learner-v0.2.0.zip
```

Or:

```bash
tar -czf nature-figure-learner-v0.2.0.tar.gz nature-figure-learner/
```

Before publishing an archive, inspect its file list and confirm it does not contain KB data, generated bytecode, private images, or backup folders.

## Versioning

Suggested versioning:

- Patch: wording fixes, small trigger clarifications.
- Minor: new workflow, schema field, query shortcut, or script feature.
- Major: breaking KB schema changes or incompatible workflow changes.

Update [CHANGELOG.md](CHANGELOG.md) for each public release.
